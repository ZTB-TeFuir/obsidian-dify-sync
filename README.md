# Obsidian-Dify-Sync

Obsidian 与 Dify 知识库双向同步插件。

## 功能概述

- 全量上传本地笔记到 Dify
- 从 Dify 拉取文档到本地
- 单文件上传
- 文件自动同步（创建/修改/删除时实时同步）

## 安装配置

1. 安装插件后，进入 **设置 -> 第三方插件 -> Dify Sync -> 设置图标**
2. 配置以下三项：

| 设置项 | 说明 | 示例 |
|--------|------|------|
| **API 基础地址** | Dify API 的基础 URL | `https://api.dify.ai/v1` |
| **API Key** | Dify 知识库的 API 密钥 | 在 Dify 知识库设置中获取 |
| **知识库 ID** | 要同步的目标知识库 ID | 在 Dify 知识库 URL 中获取 |

> API Key 获取路径：Dify -> 知识库 -> 选择目标知识库 -> API 访问 -> 创建 API Key

## 功能说明

### 1. 全量上传（侧边栏按钮）

- **操作**：点击左侧边栏的 **上传图标**（云朵+箭头）
- **逻辑**：
  - 自动区分 **新文件** 和 **已存在文件**
  - 新文件 -> 调用 Dify 创建接口
  - 已存在文件 -> 调用 Dify 更新接口
  - 完成后显示汇总，如 `上传完成：15/15 成功`
  - 如果没有需要上传的文件，提示 `没有需要上传的文件`

### 2. 从 Dify 拉取（侧边栏按钮）

- **操作**：点击左侧边栏的 **下载图标**（云朵+向下箭头）
- **逻辑**：
  - 对比 Dify 文档与本地文件，**仅拉取本地不存在的新文档**
  - 如果所有文档都已同步，提示 `全部文件均已同步，无需拉取`，不执行下载
  - 完成后显示汇总，如 `拉取完成：3/3 成功`

### 3. 上传当前文件（命令）

- **操作**：打开命令面板（`Ctrl/Cmd + P`），搜索 `上传当前文件到 Dify`
- **逻辑**：
  - 将当前活动文件上传到 Dify
  - 如果文件在 Dify 已存在 -> 更新
  - 如果文件在 Dify 不存在 -> 创建
  - 如果没有打开的文件，提示 `没有打开的文件`
  - 每次操作有单独提示，如 `已上传: 笔记.md` 或 `已更新: 笔记.md`

### 4. 自动同步（实时监听）

插件启用后会自动监听文件变化，无需手动操作：

| 事件 | 行为 | 提示 |
|------|------|------|
| **创建** `.md` 文件 | 自动上传到 Dify | `已上传: xxx.md` |
| **修改** `.md` 文件 | 自动更新 Dify 对应文档 | `已更新: xxx.md` |
| **删除** `.md` 文件 | 自动删除 Dify 对应文档 | `已从 Dify 删除文档` |

## 同步规则

- **文件名匹配**：以文件名（不含 `.md` 后缀）作为唯一标识，本地与 Dify 之间通过文件名对应
- **索引方式**：使用 `high_quality` 高质量索引，自动处理规则
- **支持格式**：仅同步 `.md`（Markdown）文件
- **子目录支持**：文件可存放在任意子目录中，按文件名匹配，不受路径影响

## 注意事项

1. **首次使用**：建议先执行一次全量上传，建立本地与 Dify 的映射关系
2. **文件重命名**：重命名文件等同于删除旧文件 + 创建新文件，Dify 上会产生新文档
3. **同名文件**：不同目录下存在同名 `.md` 文件时，会匹配到同一个 Dify 文档，建议避免同名
4. **网络问题**：批量操作时如遇网络中断，部分文件可能失败，重新执行即可，已成功的不会重复处理
5. **删除不可逆**：本地删除文件会同步删除 Dify 上的对应文档，请谨慎操作
